## How to Contribute

### 1. Report Issues
If you find bugs or have suggestions for improvement:
- Use the GitHub Issues tab
- Provide clear descriptions and steps to reproduce
- Include relevant code snippets or screenshots

### 2. Fork and Experiment
Feel free to:
- Fork this repository for your own learning

### 3. Share Feedback
I'd love to hear your thoughts on:
- Code structure and organization
- Performance optimizations
- Additional features or improvements
- Best practices I might have missed

## Code Standards

If you're submitting pull requests, please follow these guidelines:

### SQL Style
- Use CTEs for complex queries
- Add comments for business logic
- Use meaningful aliases
- Format SQL consistently

### dbt Best Practices
- Add tests for new models
- Document all columns and tables
- Use descriptive model names
- Follow the staging → mart → governance pattern

### Git Commits
- Write clear commit messages
- Keep commits focused and atomic
- Reference issues when applicable
